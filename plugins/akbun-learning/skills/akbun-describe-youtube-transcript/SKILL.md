# akbun YouTube Transcript Markdown 변환

유튜브 transcript를 Markdown 대본으로 옮기는 스킬이다. 자동 자막 중복과 추임새를 정리하고 시간 순서대로 기록한다. 기본 출력 언어는 한국어 번역이며, 사용자 요청 시 다른 언어로 번역한다.

## workflow

1. 자막과 메타데이터를 가져온다.
- 실행: `bash scripts/fetch_transcript.sh "<VIDEO_URL>" "ko,en,ja"`
- `info_json`: 영상 제목, URL, 채널, 언어, 게시일, 카테고리, 태그, 챕터
- `cleaned`: 우선 읽을 평탄화 transcript
- `subtitles`: `cleaned`가 비어 있을 때만 읽을 원본 VTT
- 자막이 없으면 더 넓은 언어 후보로 재시도하거나 사용자에게 직접 붙여넣기를 제안한다. 자막 없이 내용 생성 금지.

2. transcript를 정리한다.
- `[HH:MM:SS] 발화` 형식 사용
- rolling 중복, karaoke tag, 인라인 타이밍 태그 제거
- 음, 어, 그, 자, 뭐, uh, um, you know, like 같은 추임새 제거

3. 대본 구간을 나눈다.
- 챕터가 없으면 주제 전환, 화제 변화, 화자 변화 기준으로 나눈다.
- 챕터가 있어도 한 챕터가 5분 이상이면 2~3분 내외로 세분화한다.

4. 대본을 Markdown으로 옮긴다.
- 화자의 발화를 원문 그대로 옮긴다. 요약, 의역, 해석, 재구성 금지.
- 자동 자막의 기술적 중복과 추임새만 제거한다. 그 외 발화는 손대지 않는다.
- 기본은 한국어 번역, 사용자가 지정하면 그 언어로 번역한다. 번역도 의미를 압축하거나 더하지 않는다.
- 고유명사, 코드, 브랜드명, 숫자는 원문 표기 유지. 메타데이터에 명확한 표기가 있으면 자동 자막 오인식보다 메타데이터 우선.

화자 표기 규칙:

- 화자가 바뀔 때마다 새 문단을 시작하고 `[화자명]`을 앞에 붙인다.
- 화자명은 영상에서 이름이 언급되면 그 이름을 쓴다. 이름이 없으면 성별과 알파벳으로 구분한다(남자A, 여자A, 남자B 등).
- 화자가 1명뿐이면 화자 표기를 생략한다.

문단 나누기 규칙:

- 문맥(화제, 논점, 설명 대상, 장면)이 바뀌는 지점마다 반드시 빈 줄로 문단을 끊는다. 한 구간(H2 블록) 전체를 한 문단으로 뭉치지 않는다.
- 화자가 확실히 분리되는 영상(인터뷰, 대담, 팟캐스트 등)에서는 화자가 바뀌는 지점을 최우선 기준으로 문단을 끊는다. 같은 화자가 길게 말해도 문맥이 바뀌면 문단을 끊는다.
- 같은 문맥의 문장들은 줄바꿈 없이 한 문단으로 이어 쓴다. 문단 내부에는 줄바꿈을 넣지 않는다.
- 한 문단이 5~7문장을 넘기면 그 안에서 세부 문맥 전환점(전환 표현, 새 예시, 새 질문, 결론 진입 등)을 찾아 끊는다.
- 열거(첫 번째…두 번째…)나 단계별 설명은 항목마다 새 문단으로 나눈다.
- 한 문장이 한 문단이 되어 반복되는 패턴은 시처럼 끊어 쓴 것이므로 같은 맥락끼리 합친다.

1. 결과물을 저장하고 출력한다.
- 파일명: `$HOME/Downloads/유튜브자막-{영상제목}.md`
- 파일명 금지 문자(`/`, `\`, `:`, `*`, `?`, `"`, `<`, `>`, `|`)는 `-`로 치환한다.
- frontmatter `tags`에 `youtube/transcript`를 반드시 포함한다.
- 저장 후 절대 경로, 원본 언어, 자막 종류, 챕터 사용 여부를 안내한다.
- 같은 Markdown을 stdout에도 출력한다.

출력 템플릿:

```markdown
---
title: "{영상제목}"
url: {영상url}
channel: "{채널}"
categories: ["{분류1}", "{분류2}"]
tags:
  - youtube/transcript
language: {영상언어}
published: {YYYY-MM-DD}
---

# 요약

- 영상 전체를 아우르는 요약을 최대 5문장으로 작성한다.
- 자막에 없는 해석이나 설명은 추가하지 않는다. 전체 요약이 어려우면 챕터별로 1~2문장씩 쓴다.
- 대본과 동일한 문단 규칙을 따른다.

# 질문지

영상 내용을 깊이 이해하는 데 도움이 되는 질문을 5개 미만으로 제시한다.

# 대본

## [HH:MM:SS~HH:MM:SS] {챕터 또는 구간 라벨 1}

[남자A] 화자의 발화를 원문 그대로 적는다. 같은 맥락 문장들은 줄바꿈 없이 이어 쓴다.

[여자A] 다른 화자가 말하면 새 문단으로 나눈다.

[남자A] 열거나 단계별 설명은 항목마다 새 문단으로 나눈다.

## [HH:MM:SS~HH:MM:SS] {챕터 또는 구간 라벨 2}

{...}
```

## 저장 전 체크리스트

- 문단: 문맥이 바뀌는 지점마다 문단이 끊겨 있는가? 한 구간이 통째로 한 문단이 되지 않았는가? 한 문장 한 문단 반복도 없는가?
- 화자 분리: 화자가 확실히 구분되는 영상에서 화자 단위로 문단이 끊겨 있는가?
- 파일명/태그: `유튜브자막-`로 시작하고 frontmatter `tags`에 `youtube/transcript`가 있는가?
- 원문 충실성: 요약, 의역, 해석 없이 화자의 발화를 그대로 옮겼는가?
- 화자 표기: 화자가 바뀔 때마다 `[화자명]`으로 새 문단을 시작했는가?
- 구간: 블록이 2~3분 내외로 나뉘었는가?
- 충실성: 핵심 숫자, 고유명사, 발화 내용이 훼손되지 않았는가?
- 구조: H1 섹션(`# 요약`, `# 질문지`, `# 대본`), H2 타임라인 블록(`## [HH:MM:SS~HH:MM:SS] 라벨`) 구조를 지켰는가?
