# akbun YouTube 자막 풀어쓰기

한국어 독자가 유튜브 영상을 보지 않고도 **화자가 실제로 한 말**을 이해할 수 있도록 돕는 스킬이다. 출력물은 영상의 자막을 토대로 다시 정리한 한국어 노트이며, **짧은 요약이 아니라 자막에 충실한 풀어쓰기**가 목적이다. `## 요약` 섹션은 본문을 읽기 전에 방향을 잡아주는 짧은 블록이고, 본체는 타임라인을 따라 화자의 말을 옮긴 `## 타임라인` 섹션이다. 영상이 영어든 일본어든 출력은 기본적으로 한국어로 한다.

## 이 스킬을 실행하는 시점

사용자가 유튜브 링크(또는 video id)를 주면서 내용을 설명/풀어쓰기/정리/필기해 달라고 요청할 때 실행한다. 사용자가 링크만 던지고 별다른 지시가 없으면 한 번 정도 "자막 기반으로 풀어쓸까요?" 라고 확인한 뒤 진행한다. 명확하게 요청한 경우에는 곧장 1단계로 넘어간다.

## 1단계 — 자막과 메타데이터 받기

번들된 부트스트랩 스크립트를 실행한다. 처음 실행 시 `~/.cache/akbun-youtube-summary/venv` 안에 yt-dlp를 자동 설치하므로 사용자는 별도의 전역 설치를 할 필요가 없다. 스크립트는 다음 두 파일을 받아 임시 디렉터리에 둔다.

- `*.info.json` — 영상 메타데이터 (제목, 채널, 게시일, 챕터, 카테고리, 태그, 언어 등).
- `*.{lang}.vtt` — 사용자가 우선순위로 지정한 언어의 자막 원본. SRT가 아닌 VTT를 그대로 둔다. yt-dlp의 SRT 변환기는 `ffmpeg`를 요구하기 때문에 사용자에게 추가 설치를 강요하지 않으려고 VTT를 유지한다. VTT도 `HH:MM:SS.mmm` 타임스탬프를 그대로 담고 있다.
- `*.{lang}.cleaned.txt` — VTT를 후처리한 평탄화 자막. 카라오케 단어-타이밍 태그(`<00:00:15.440><c>...</c>`)를 제거하고, 유튜브 자동 자막의 "롤링" 중복(직전 큐의 꼬리가 다음 큐의 머리에 그대로 다시 들어가는 패턴)을 합쳐서 한 줄에 하나씩 `[HH:MM:SS] 새로 추가된 발화` 형태로 떨어뜨려 둔다. **읽을 때는 이 파일을 우선 사용한다** — 10분짜리 영상의 원본 VTT는 25,000 토큰을 넘어 `Read` 한도를 깨는 일이 흔하지만, 평탄화 자막은 보통 그 1/4 ~ 1/5 크기다.

스크립트 실행 예시:

```bash
bash scripts/fetch_transcript.sh "<VIDEO_URL>" "ko,en,ja"
```

스크립트는 한 줄짜리 JSON을 stdout으로 출력한다: `{"workdir": "...", "info_json": "...", "subtitles": "...", "cleaned": "..."}`. 임시 파일은 macOS/Linux에서 `/tmp/akbun-youtube-summary/<timestamp>-<pid>/`, Windows에서 `%TEMP%\akbun-youtube-summary\...` 아래에 들어간다. 임시 영역이므로 사용자가 따로 정리하지 않아도 된다.

`subtitles` 값이 빈 문자열이면 해당 언어들에 대해 정식 자막도 자동 생성 자막도 없는 상태다. 사용자에게 그 사실을 알리고, 더 넓은 언어 후보로 재시도(`bash scripts/fetch_transcript.sh "URL" "ko,en,ja,zh,es,fr,de,ru,pt"`)하거나 사용자가 직접 자막을 붙여넣는 방식을 제안한다. **자막이 없는 상태에서 임의로 내용을 지어내지 않는다.**

연령 제한이나 로그인 필요 영상은 스크립트가 실패할 수 있다. 그럴 때는 브라우저 쿠키를 사용해 직접 실행한다. 예: `~/.cache/akbun-youtube-summary/venv/bin/yt-dlp --cookies-from-browser chrome --skip-download --write-auto-subs --sub-langs ko,en --sub-format vtt -o "/tmp/%(title)s.%(ext)s" "URL"`.

## 2단계 — 메타데이터와 자막 읽기

스크립트가 알려준 `info_json`과 `cleaned` 경로를 `Read` 도구로 읽는다. `cleaned`가 비어 있을 때만 `subtitles`(원본 VTT)로 폴백한다. 메타데이터에서 다음 필드를 뽑는다.

| 필드 | info.json의 출처 |
|:--|:--|
| 영상제목 | `title` |
| 영상url | `webpage_url` |
| 영상분류 | `categories` + `tags` (합치고 중복 제거, 8개 이하로 줄임) |
| 영상언어 | `language` (없으면 VTT 파일 이름의 언어 코드, 예: `*.en.vtt` → 영어) |
| 영상게시날짜 | `upload_date` (`YYYYMMDD` → `YYYY-MM-DD`) |
| 채널 | `uploader` |
| chapters | `chapters` 배열 (`{start_time, end_time, title}`). 비어 있을 수 있다 |

평탄화 자막(`cleaned.txt`)은 한 줄에 한 큐 형태로 `[HH:MM:SS] 발화 텍스트`가 시간 순서로 들어 있다. 유튜브 자동 자막의 카라오케 단어 타이밍 태그와 롤링 중복은 후처리 단계에서 이미 제거되어 있으므로, 모델은 이 파일을 그대로 읽고 각 줄의 타임스탬프를 출력 섹션의 시간 앵커로 사용하면 된다. 폴백으로 원본 VTT(`subtitles`)를 읽어야 하는 상황이라면, `WEBVTT` 헤더 아래 `HH:MM:SS.mmm --> HH:MM:SS.mmm` 큐가 반복되는 형식이고, 직전 큐의 꼬리가 다음 큐의 머리에 그대로 다시 들어가는 롤링 중복과 `<00:00:15.440><c> hello</c>` 같은 인라인 타이밍 태그는 직접 무시하면서 읽어야 한다.

## 3단계 — 타임라인 구조 결정

이 결정이 출력 전체의 모양을 좌우한다.

1. **`chapters`가 비어 있지 않으면**, 챕터 제목이 보고서의 섹션 헤더가 되고 각 섹션의 시작/끝 타임스탬프는 챕터의 `start_time` / `end_time`에서 그대로 가져온다. 제작자의 챕터를 신뢰한다 — 임의로 쪼개거나 합치지 않는다.
2. **`chapters`가 비어 있으면**, 자막의 자연스러운 주제 전환점(새로운 화제, 명백한 전환 표현, 다른 화자 블록 등)을 보고 4~8개 섹션을 직접 추론한다. 시작/끝 타임스탬프는 주변 자막 큐에서 가져온다. 섹션 길이는 비슷하게 맞추되, 10분 영상이 4섹션, 60분 영상이 7섹션 정도여도 무방하다.

모든 섹션 헤더는 `[HH:MM:SS~HH:MM:SS] 주제` 형식으로 적는다. 두 자리 패딩과 양방향 타임스탬프를 항상 유지한다.

## 4단계 — 화자의 말을 정리해서 옮기기

사용자는 화자의 말을 **그대로** 받기를 원한다. 다만 정돈해서. 각 섹션 본문을 쓸 때 다음 필터를 적용한다.

- **추임새 제거**: 음, 어, 그, 자, 뭐, 그니까, uh, um, you know, like(추임새 용도), eh, ah, kind of(추임새 용도), sort of(추임새 용도) 등. 추임새 자체가 인용의 핵심일 때만 살린다.
- **농담과 곁가지 제거**: 주제와 무관한 잡담(날씨 이야기, 광고 문구의 농담조 표현, 진행자끼리의 잡담, "오늘 촬영 힘드네요" 같은 자기참조 등)은 뺀다.
- **화자의 문장과 논리는 보존**: 자기 말로 요약하지 말고 화자가 한 말을 그대로 옮긴다(다만 추임새/농담은 제거). 독자는 깔끔히 다듬은 자막을 읽는 느낌을 받아야 하지, 풀어쓴 사람의 해설을 읽는 느낌을 받으면 안 된다.
- **한국어로 번역**: 영어/일본어 등 원본 언어가 한국어가 아니면 한국어(외국어표기법)로 충실히 옮긴다. 해석하지 말고 번역하지 말고 그대로 옮긴다. 영상에 등장하는 고유명사, 숫자, 예시는 그대로 유지한다(예: `$10`을 `₩13,000`으로 환산하지 않는다).
- **고유명사 표기는 메타데이터를 우선**: 자동 생성 자막에는 고유명사 오인식이 잦다(예: `Adrien` → `Adria`). 영상 제목, 채널명, 영상 설명 같은 메타데이터에 같은 이름의 분명한 표기가 있으면 그쪽을 따른다. 이는 화자의 말을 임의로 바꾸는 것이 아니라, 자막의 transcription error를 메타데이터가 가진 정답으로 교정하는 작업이다. 단, 메타데이터에도 근거가 없으면 자막 표기를 그대로 둔다.

각 섹션 본문에서 "이 섹션에서 가장 중요한 한 문장만 인용한다면?"이라고 물었을 때 인용할 만한 1~3개 어구는 `**bold**`로 강조한다. 본문 절반을 굵게 칠하지 말고, 굵게 표시한 부분이 진짜로 중요한 신호가 되도록 아껴 쓴다.

## 5단계 — 출력 만들기

결과 파일은 `$HOME/Downloads/{영상제목}.md`에 저장한다. 파일명은 안전하게 다듬는다: `/`, `\`, `:`, `*`, `?`, `"`, `<`, `>`, `|` 는 모두 `-`로 치환하고, 끝의 공백과 점은 잘라내며, 길이는 약 120자 이하로 제한한다. 같은 마크다운을 stdout에도 출력해서 사용자가 채팅에서 바로 읽을 수 있게 한다.

### 출력 템플릿

```markdown
---
title: "{영상제목}"
url: {영상url}
channel: "{채널}"
categories: [{영상분류 — 따옴표로 감싼 문자열을 콤마로 구분}]
language: {영상언어}
published: {YYYY-MM-DD}
---

# {영상제목}

## 영상의 의도

{1~3 문장. 영상 제작자가 이 영상으로 시청자에게 전달하려는 핵심 의도. 시청자가 무엇을 얻기를 원하는지를 짧게 표현한다.}

## 요약

{5문장 이하의 한국어 요약. 본 보고서의 목적은 요약이 아니라 자막 기반의 풀어쓰기이므로, 이 섹션은 본문 타임라인을 읽기 전 잠깐 방향을 잡아주는 용도로만 짧게 쓴다. 절대 6문장 이상 쓰지 않는다.}

## 인사이트와 질문

이 영상을 본 사람이 다음 단계에서 던질 만한 질문 / 곱씹을 만한 인사이트 10개. **인사이트 5개 + 질문 5개의 균형을 권장**한다.

1. {인사이트 또는 질문}
2. ...
10. ...

## 타임라인

이 섹션이 보고서의 본체다. 화자가 실제로 말한 내용을 시간 순서대로 풀어쓴다 — 짧게 요약하지 않는다. 한 섹션당 보통 3~10문장.

[HH:MM:SS~HH:MM:SS] {주제 1}
{화자의 대사를 추임새/농담만 제거한 형태로 한국어로 옮긴 본문. 화자가 든 예시, 숫자, 이름은 그대로 유지하고, 화자의 논리 흐름을 압축하지 말고 따라간다. 중요한 부분은 **bold**.}

[HH:MM:SS~HH:MM:SS] {주제 2}
{...}
```

템플릿 보충 설명:

- frontmatter 필드는 정확히 `title`, `url`, `channel`, `categories`, `language`, `published`이다. 한국어 라벨은 이 YAML 키로 매핑되어 파일이 기계적으로도 파싱 가능하다.
- `## 타임라인` 섹션의 각 블록은 마크다운 서브헤더(`###`)를 쓰지 **않는다**. `[HH:MM:SS~HH:MM:SS] 주제` 줄은 그냥 평문으로 한 줄에 두고, 그 아래 줄에 본문을 단락으로 쓴다. 사용자 명세를 그대로 따른다.
- 각 섹션 본문은 최소 한 문장 이상이다. 챕터가 너무 짧아서 추임새/농담을 걷어내면 의미 있는 내용이 없다 해도, 화자가 그 구간에서 어떤 주제를 다뤘다는 한 문장은 반드시 적는다 — 빈 채로 두지 않는다.

## 6단계 — 사용자에게 결과 알리기

파일을 저장한 뒤 한국어로 다음을 알려준다: 저장된 마크다운의 절대 경로, 감지된 원본 언어, 자막 종류(파일명이 `*.{lang}.vtt`이면서 `info.json`의 `subtitles` 키에 들어 있던 것이면 정식 업로드 자막, `automatic_captions`에서 가져온 것이면 자동 생성 자막. 자동 생성 자막은 정확도가 낮을 수 있으니 그 한계도 함께 안내), 그리고 제작자의 챕터를 사용했는지 직접 추론했는지. 그 다음 마크다운 본문 전체를 stdout에 그대로 덤프해 사용자가 채팅에서 바로 읽도록 한다.

## 가드레일

- **출력은 기본 한국어**. 영상이 영어나 일본어여도 보고서는 한국어로 만든다. 사용자가 명시적으로 다른 언어를 요청한 경우(예: "영어로 정리해줘", "in English please")에만 언어를 바꾼다.
- **없는 내용을 지어내지 않는다**. 자막이 없거나 스크립트가 실패하거나 어떤 섹션에 의미 있는 발화가 정말로 없으면 그 사실을 그대로 적는다. 충실한 짧은 노트가 상상으로 채운 긴 노트보다 낫다.
- **풀어쓰는 사람의 목소리를 화자의 자리에 끼워 넣지 않는다**. `## 타임라인`은 정돈한 자막이지 해설이 아니다. 화자가 사실관계를 틀렸어도 그대로 옮긴다(따로 바로잡지 않는다). 정말로 짚어야 할 부분이 있다면 보고서 맨 끝에 `## 메모` 섹션을 한 번만 짧게 추가할 수 있지만, 자주 쓰지 않는다.
- **고유명사, 브랜드명, 코드, 숫자는 번역하지 않는다**. 원본 그대로 둔다. 한국어 번역은 그 주변 문장에만 적용한다.
- **파일명 안전성**. 영상 제목에 운영체제에서 금지된 문자가 들어 있으면 5단계에 적힌 대로 다듬되, 사람이 읽을 수 있는 형태는 유지한다. 모든 글자를 하이픈으로 치환하지 않는다.
- **한 영상에 한 보고서**. 사용자가 재생목록 URL을 주면 어떤 영상을 원하는지 묻거나, 각 영상을 별도의 파일로 만든다. 재생목록 전체를 한 보고서에 욱여넣지 않는다.
