# akbun YouTube Transcript Markdown 변환

유튜브 transcript를 Markdown 대본으로 옮기는 스킬이다. 유튜브 자막에 있는 내용을 시간 순서대로 가져오고, 자동 자막 중복과 추임새를 정리한다. 기본 출력 언어는 한국어 번역이며, 사용자가 요청하면 영어, 일본어 등 다른 언어로 번역할 수 있다.

## workflow

1. 자막과 메타데이터를 가져온다.

- 실행: `bash scripts/fetch_transcript.sh "<VIDEO_URL>" "ko,en,ja"`
- `info_json`: 영상 제목, URL, 채널, 언어, 게시일, 카테고리, 태그, 챕터
- `cleaned`: 우선 읽을 평탄화 transcript
- `subtitles`: `cleaned`가 비어 있을 때만 읽을 원본 VTT
- 자막이 없으면:
  - 더 넓은 언어 후보로 재시도
  - 사용자가 직접 자막을 붙여넣는 방식 제안
  - 자막 없이 내용 생성 금지

2. transcript를 읽고 정리한다.

- `cleaned`: `[HH:MM:SS] 발화` 형식
- 원본 VTT만 있으면 cue 타임스탬프 사용
- rolling 중복, karaoke tag, 인라인 타이밍 태그 제거
- 음, 어, 그, 자, 뭐, uh, um, you know, like 같은 추임새 제거

3. 대본 구간을 나눈다.

- 챕터가 없으면 주제 전환, 화제 변화, 화자 변화 기준
- 챕터가 있어도 한 챕터가 5분 이상이면 2~3분 내외로 세분화
- 모든 블록은 `[HH:MM:SS~HH:MM:SS] 챕터 또는 구간 라벨` 형식

4. 대본을 Markdown으로 옮긴다.

- 화자가 말한 내용을 빠뜨리지 않음
- 자동 자막의 기술적 중복과 추임새만 제거
- 발화의 의미를 임의로 덜어내거나 덧붙이지 않음
- 기본은 한국어 번역
- 사용자가 출력 언어를 지정하면 그 언어로 번역
- 고유명사, 코드, 브랜드명, 숫자는 원문 표기 유지
- 메타데이터에 분명한 고유명사 표기가 있으면 자동 자막의 오인식보다 메타데이터 우선

5. 결과물을 저장하고 출력한다.

- 경로: `$HOME/Downloads/{영상제목}.md`
- 파일명 금지 문자(`/`, `\`, `:`, `*`, `?`, `"`, `<`, `>`, `|`)는 `-`로 치환
- 저장 후 절대 경로, 원본 언어, 자막 종류, 챕터 사용 여부 안내
- 같은 Markdown을 stdout에도 출력
- 출력 템플릿

```markdown
---
title: "{영상제목}"
url: {영상url}
channel: "{채널}"
categories: ["{분류1}", "{분류2}"]
language: {영상언어}
published: {YYYY-MM-DD}
---

# {영상제목}

## 요약

- 영상의 화자가 말하고자 하는 의도를 최대 5문장 미만으로 요약한다.
- 영상 제목과 챕터 라벨을 참고하되, 자막에 없는 설명이나 해석은 추가하지 않는다. 영상 전체를 아우르는 요약이 어려울 때는 챕터별로 1~2문장씩 요약한다.
- 요약문은 영상의 실제 발화에 충실하게 작성한다.
- 문장은 맥락이 다르면 개행으로 문단을 나눈다.

## 질문지

영상의 화자가 말하고자 하는 의도를 파악하는 데 도움이 되는 질문을 5개 미만으로 제시한다.

## 대본

[HH:MM:SS~HH:MM:SS] {챕터 또는 구간 라벨 1}
{자막에 있는 화자의 발화를 출력 언어로 옮긴다. 기술적 중복과 추임새만 제거한다.}

{의미 단위가 바뀌면 이렇게 새 문단으로 나눈다.}

[HH:MM:SS~HH:MM:SS] {챕터 또는 구간 라벨 2}
{...}
```

`## 대본`의 각 블록은 `###` 헤더로 만들지 않는다. `[HH:MM:SS~HH:MM:SS] 챕터 또는 구간 라벨` 줄을 평문으로 두고, 바로 아래에 대본을 쓴다. 각 블록에는 최소 한 문장 이상을 넣는다.

## 산출물 저장 전 평가 요소

저장하기 전에 아래 조건을 스스로 확인하고, 하나라도 실패하면 결과물을 고친 뒤 저장한다.

- 문단 나누기: 대본이 의미 단위로 나뉘었는가?
- 어조: 화자의 말투와 의미를 임의로 바꾸지 않았는가?
- 구간 세분화: 블록이 2~3분 내외로 촘촘하게 나뉘었는가?
- 내용 충실성: 화자의 실제 발화, 핵심 숫자, 고유명사가 훼손되지 않았는가?
- 무해석성: 자막에 없는 설명, 요약, 평가, 질문이 섞이지 않았는가?
- 템플릿 정합성: frontmatter와 Markdown 구조를 그대로 지켰는가?
