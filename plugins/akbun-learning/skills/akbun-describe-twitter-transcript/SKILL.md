# akbun X(Twitter) Video Transcript Markdown 변환

x.com post에 첨부된 영상의 transcript를 Markdown 대본으로 옮기는 스킬이다. 자막 트랙이 있으면 그대로 쓰고, 없으면 로컬 whisper 전사로 대본을 만든다. 기본 출력 언어는 한국어 번역이며, 사용자 요청 시 다른 언어로 번역한다.

## workflow

1. 자막과 메타데이터를 가져온다.

- 실행: `bash scripts/fetch_transcript.sh "<X_POST_URL>" "ko,en,ja"`
- `info_json`: 트윗 본문(`description`), 작성자(`uploader`, `uploader_id`), 게시일, 영상 길이
- `cleaned`: 우선 읽을 평탄화 transcript
- `subtitles`: `cleaned`가 비어 있을 때만 읽을 원본 VTT
- `source`: transcript 출처. `subtitles`(자막 트랙) 또는 `whisper`(로컬 전사)
- whisper 전사는 첫 실행 시 모델 다운로드로 수 분 걸릴 수 있다. 시간이 걸린다고 중단하지 않는다.
- `source`가 `none`이면(자막도 전사도 실패) 사용자에게 실패 원인을 알리고 직접 붙여넣기를 제안한다. transcript 없이 내용 생성 금지.
- 비공개 계정이나 로그인 필요 영상은 접근 불가로 안내한다.

1. transcript를 정리한다.

- `[HH:MM:SS] 발화` 형식 사용
- rolling 중복, karaoke tag, 인라인 타이밍 태그 제거
- 음, 어, 그, 자, 뭐, uh, um, you know, like 같은 추임새 제거
- `source`가 `whisper`이면 오인식 가능성이 있다. 고유명사와 숫자는 트윗 본문(`description`)의 표기를 우선한다.

1. 대본 구간을 나눈다.

- X 영상은 챕터가 없으므로 주제 전환, 화제 변화, 화자 변화 기준으로 나눈다.
- 영상이 2분 이하로 짧으면 구간을 나누지 않고 하나의 블록으로 둔다.
- 길면 2~3분 내외로 세분화한다.

1. 대본을 Markdown으로 옮긴다.

- 화자의 발화를 원문 그대로 옮긴다. 요약, 의역, 해석, 재구성 금지.
- 자동 전사의 기술적 중복과 추임새만 제거한다. 그 외 발화는 손대지 않는다.
- 기본은 한국어 번역, 사용자가 지정하면 그 언어로 번역한다. 번역도 의미를 압축하거나 더하지 않는다.
- 고유명사, 코드, 브랜드명, 숫자는 원문 표기 유지. 트윗 본문에 명확한 표기가 있으면 전사 오인식보다 트윗 본문 우선.

화자 표기 규칙:

- 화자가 바뀔 때마다 새 문단을 시작하고 `[화자명]`을 앞에 붙인다.
- 화자명은 영상에서 이름이 언급되면 그 이름을 쓴다. 이름이 없으면 성별과 알파벳으로 구분한다(남자A, 여자A, 남자B 등).
- 화자가 1명뿐이면 화자 표기를 생략한다.

문단 나누기 규칙:

- 한 문단은 같은 맥락(화제, 설명, 장면)을 다루는 문장들을 줄바꿈 없이 이어 쓴다.
- 화제가 바뀔 때만 빈 줄로 새 문단을 시작한다. 문단 내부에는 줄바꿈을 넣지 않는다.
- 열거(첫 번째…두 번째…)나 단계별 설명은 항목마다 새 문단으로 나눈다.
- 한 문장이 한 문단이 되어 반복되는 패턴은 시처럼 끊어 쓴 것이므로 같은 맥락끼리 합친다.

1. 결과물을 저장하고 출력한다.

- 파일명: `$HOME/Downloads/트위터자막-{제목}.md`
- 제목은 트윗 본문 첫 문장에서 만든다. 본문이 없거나 너무 길면 영상 내용을 대표하는 짧은 제목을 만든다.
- 파일명 금지 문자(`/`, `\`, `:`, `*`, `?`, `"`, `<`, `>`, `|`)는 `-`로 치환한다.
- frontmatter `tags`에 `twitter/transcript`를 반드시 포함한다.
- 저장 후 절대 경로, 원본 언어, transcript 출처(자막 트랙/whisper 전사)를 안내한다.
- 같은 Markdown을 stdout에도 출력한다.

출력 템플릿:

```markdown
---
title: "{제목}"
url: {post url}
author: "{작성자 표시명 (@계정)}"
tags:
  - twitter/transcript
language: {영상언어}
source: {subtitles 또는 whisper}
published: {YYYY-MM-DD}
---

# 트윗 본문

트윗 본문을 원문 그대로 적고, 원문이 한국어가 아니면 아래에 한국어 번역을 덧붙인다.

# 요약

- 영상 전체를 아우르는 요약을 최대 5문장으로 작성한다.
- transcript에 없는 해석이나 설명은 추가하지 않는다.
- 대본과 동일한 문단 규칙을 따른다.

# 질문지

영상 내용을 깊이 이해하는 데 도움이 되는 질문을 5개 미만으로 제시한다.

# 대본

## [HH:MM:SS~HH:MM:SS] {구간 라벨 1}

[남자A] 화자의 발화를 원문 그대로 적는다. 같은 맥락 문장들은 줄바꿈 없이 이어 쓴다.

[여자A] 다른 화자가 말하면 새 문단으로 나눈다.

## [HH:MM:SS~HH:MM:SS] {구간 라벨 2}

{...}
```

## 저장 전 체크리스트

- 문단: 한 문장 한 문단이 반복되지 않는가? 빈 줄은 화제가 바뀌는 지점과 열거 항목 사이에만 있는가?
- 파일명/태그: `트위터자막-`로 시작하고 frontmatter `tags`에 `twitter/transcript`가 있는가?
- 원문 충실성: 요약, 의역, 해석 없이 화자의 발화를 그대로 옮겼는가?
- 화자 표기: 화자가 바뀔 때마다 `[화자명]`으로 새 문단을 시작했는가?
- 구간: 2분 초과 영상의 블록이 2~3분 내외로 나뉘었는가?
- 충실성: 핵심 숫자, 고유명사, 발화 내용이 훼손되지 않았는가? whisper 전사면 트윗 본문 표기와 대조했는가?
- 구조: H1 섹션(`# 트윗 본문`, `# 요약`, `# 질문지`, `# 대본`), H2 타임라인 블록(`## [HH:MM:SS~HH:MM:SS] 라벨`) 구조를 지켰는가?
