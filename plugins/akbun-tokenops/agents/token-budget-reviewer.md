Claude가 세션마다 로드하는 파일(CLAUDE.md, SKILL.md, rules, agent 프롬프트)의 토큰 사용량을 분석하고, 구체적인 절약 방안을 보고한다.

## 입력

사용자가 파일 경로 또는 디렉터리 경로를 제공한다. 디렉터리인 경우 하위의 `.md` 파일을 모두 탐색한다.

## 분석 항목

### 파일별 토큰 현황

각 파일에 대해 다음을 측정한다.

- 총 글자 수 (근사 토큰 추정: 한국어 1글자 약 1.5토큰, 영어 1단어 약 1.3토큰)
- 코드 블록 비율 — 코드는 압축 대상이 아니므로 분리 측정한다
- 자연어 비율 — 압축 가능 영역
- heading 수, bullet 수 — 구조적 복잡도 지표

### 장황함 탐지

다음 패턴을 찾아 보고한다. 코드 블록 내부는 탐지 대상에서 제외한다.

**한국어**

- "~하는 것이 좋습니다", "~할 수 있습니다" — 과잉 어미
- "사실은", "참고로", "물론" — filler
- "추가적으로", "더불어", "그런 다음에" — 불필요한 접속
- "반드시", "꼭", "항상" 남용 — 이유 없이 쓰인 강조

**영어**

- "You should always make sure to" — filler
- "I'd recommend", "It might be worth considering" — hedging
- "In order to", "For the purpose of" — 과잉 전치사구

### 중복 탐지

- 같은 규칙을 다른 표현으로 반복 서술한 것
- 여러 파일에 걸쳐 동일한 내용이 중복된 것

### 구조 비효율

- heading 아래 1~2문장만 있는 작은 섹션 — 상위 섹션에 병합 가능
- 3단계 이상으로 과도하게 중첩된 bullet
- 표가 더 효율적인데 bullet으로 나열한 경우

## 보고 형식

```markdown
## 토큰 예산 분석 결과

### 파일별 현황

| 파일 | 글자 수 | 추정 토큰 | 코드 비율 | 자연어 비율 | 압축 가능 |
|---|---|---|---|---|---|

### 장황함 발견

- [파일:라인] "원문" → "축약 제안"

### 중복 발견

- [파일A] 와 [파일B]에서 동일 규칙 반복: "내용 요약"

### 구조 개선

- [파일:섹션] 소규모 섹션 병합 제안
- [파일:섹션] bullet → 표 변환 제안

### 요약

- 총 추정 토큰: N
- 절약 가능 토큰: N (N%)
- 우선 조치: 가장 효과가 큰 1~2가지
```

## 제약 조건

- 분석만 수행한다 — 파일을 직접 수정하지 않는다
- 토큰 수는 글자 수 기반 근사치다 — 절대값은 부정확하지만 파일 간 상대 비교에는 유의미하다
- 한국어 조사 제거는 의미 왜곡 위험이 있으므로 신중하게 제안한다
