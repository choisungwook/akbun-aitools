한국어 자연어 파일(CLAUDE.md, memory, 규칙 파일)을 압축해서 입력 토큰을 줄인다. 압축 파일이 원본을 덮어쓰고, 원본은 `<파일명>.original.md`로 백업한다.

## Trigger

`/akbun-compress-ko <filepath>` 또는 사용자가 파일 압축을 요청할 때.

## 워크플로

1. 파일을 읽는다
2. 자연어인지 확인한다 — `.md`, `.txt`만 대상이다. 코드 파일은 건드리지 않는다
3. 백업 파일(`<파일명>.original.md`)이 이미 존재하면 중단한다 — 기존 백업을 덮어쓰면 원본을 잃기 때문이다
4. 원본을 `<파일명>.original.md`로 백업한다
5. 아래 규칙에 따라 압축한다
6. 압축 결과를 원본 경로에 덮어쓴다
7. 결과를 보고한다

## 압축 규칙

### 제거 대상

- **한국어 filler:** "사실", "참고로", "물론", "아무튼", "그러니까", "말하자면", "간단히 말하면"
- **과잉 어미:** "~하는 것이 좋습니다" → "~한다", "~할 수 있습니다" → "가능하다", "~하는 것을 권장합니다" → "권장한다"
- **불필요한 접속:** "그리고 나서", "그런 다음에", "추가적으로", "더불어", "또한"
- **hedging:** "아마도", "혹시", "어쩌면", "~일 수도 있습니다"
- **인사말, 맺음말:** "안녕하세요", "도움이 되셨으면", "궁금한 점이 있으시면"
- **중복 표현:** "반드시 꼭" → "반드시", "미리 사전에" → "사전에"

### 수정하지 않는 것

다음 요소는 압축 대상에서 제외한다 — 한 글자라도 바뀌면 코드나 참조가 깨지기 때문이다.

- 코드 블록 (fenced ``` 와 indented)
- inline code (`backtick` 내용)
- URL, markdown link
- 파일 경로 (`/src/components/...`, `./config.yaml`)
- 명령어 (`kubectl apply`, `git commit`, `docker build`)
- 기술 용어 (라이브러리명, API명, 프로토콜, 알고리즘)
- 고유명사 (프로젝트명, 인명, 회사명)
- 날짜, 버전 번호, 숫자
- 환경 변수 (`$HOME`, `NODE_ENV`)
- YAML frontmatter

### 구조 보존

- markdown heading — heading 텍스트를 유지하고 본문만 압축한다
- bullet 목록 계층 — nesting level을 유지한다
- 번호 목록 — 번호를 유지한다
- 표 — 셀 텍스트를 압축하되 구조를 유지한다

### 한국어 고유 규칙

한국어는 조사가 의미를 결정한다. 조사를 함부로 빼면 뜻이 달라지므로 안전한 축약만 적용한다.

**안전한 축약:**

- "~하는 것" → "~함"
- "~하기 위해서" → "~위해"
- "~에 대해서" → "~에 대해"
- "~하도록 한다" → "~한다"
- "~라고 할 수 있다" → "~이다"

**위험한 축약 (적용하지 않는다):**

- 주격 조사 "이/가" 제거 — 주어가 불분명해진다
- 목적격 조사 "을/를" 제거 — 목적어가 불분명해진다
- 처소격 "에서" → "에" — 장소와 방향의 의미가 달라진다

## 예시

압축 전:

```text
이 프로젝트에서는 Kubernetes 클러스터를 구성할 때 반드시 `kubeadm`을 사용하는 것을 권장합니다.
그 이유는 `kubeadm`이 클러스터 초기화 과정을 자동화해주기 때문입니다.
추가적으로, 노드를 추가할 때는 `kubeadm join` 명령어를 사용하시면 됩니다.
```

압축 후:

```text
Kubernetes 클러스터 구성 시 `kubeadm` 사용을 권장한다.
`kubeadm`이 클러스터 초기화를 자동화한다.
노드 추가는 `kubeadm join`으로 한다.
```

## 경계

- 자연어 파일만 압축한다 — `.py`, `.js`, `.ts`, `.json`, `.yaml`, `.yml` 등 코드/설정 파일은 건드리지 않는다
- `<파일명>.original.md`는 압축하지 않는다 — 백업 파일을 다시 압축하면 원본을 잃는다
- 혼합 파일(산문 + 코드)은 산문 부분만 압축한다
- 압축 후 heading 수, code block, URL이 원본과 다르면 실패로 판정하고 원본을 복원한다
