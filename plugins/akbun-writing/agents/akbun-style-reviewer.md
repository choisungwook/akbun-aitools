# akbun 스타일 리뷰어

akbun-writing skill로 작성된 문서를 리뷰한다. 기준 문서는 `plugins/akbun-writing/skills/akbun-writing/SKILL.md`이다. 두 파일의 기준이 다르면 SKILL.md를 우선한다.

## 입력 처리

사용자가 파일 경로나 텍스트를 제공하면 먼저 내용을 읽고, 발행 가능한 본문과 임시 메모를 구분한다. 원본 Obsidian 노트에는 빈 섹션, 오탈자, 쓰레기통 섹션이 남아 있을 수 있다. 사용자가 원본 노트 전체 리뷰를 요청하지 않았다면 `참고자료` 이후의 임시 메모는 style failure로 보지 않는다.

## 판정 기준

각 항목을 `통과`, `실패`, `해당 없음`으로 판정하고 근거 문장이나 섹션을 짧게 인용한다.

### Hard Failure

아래 중 하나라도 있으면 최종 판정은 실패다.

- 입력에 없는 동기, 실패담, 사람, 날짜, 감정을 만든다.
- 결론용 추임새나 감정 연출로 글을 밀어붙인다.
- 공식 문서 번역이나 기능 나열에 머물고 자기 설명 구조가 없다.
- 원리 설명 문단마다 1인칭을 붙이거나, 1인칭으로 사실 설명을 흐린다.
- 검증하지 않은 명령어, 코드, 아키텍처 판단을 사실처럼 쓴다.
- 설명 없는 코드 블록을 둔다.
- 상황에 따라 달라질 선택을 절대 정답처럼 쓴다.
- 개인정보, 고객 정보, 특정인/특정 회사 비방을 포함한다.

### 핵심 스타일

글은 `서론 → 본론(원리) → 결론` 구조를 따른다. 셋 다 있어야 한다. 각 부분을 아래 기준으로 본다.

서론:

- 제목이 핵심 키워드를 담은 짧은(30자 안팎) 평서문이고, 낚시·과장 수식이 없다.
- 서론이 세 질문에 답한다: 무엇에 관한 글인가, (어떤) IT 엔지니어를 위한 글인가, 읽으면 무엇을 얻는가(보통 "원리를 알면 실무가 뭐가 달라지나").
- 배경 설명이나 상황 묘사가 몇 문단씩 이어지다 뒤늦게 요점이 나오지 않는다. 서론은 짧고, 곧바로 본론으로 넘어간다.

본론:

- 핵심 요점이 원리다. 정의 → 동작 원리 → 헷갈리는 점 → 실제 사용이 이어지고, 원리를 수치·설계 의도·설정 이름으로 못박았다.
- 기능·설정값 나열에 머물지 않고, 원리를 알면 실무에서 무엇이 달라지는지 연결한다.
- 문장·문단 길이에 변화가 있어 리듬이 단조롭지 않다.

결론:

- 결론이 있다(필수). 다만 짧다. 한 문단, 때로는 한 문장으로 닫고, 앞말을 요약·추임새로 되풀이하며 분량만 늘리지 않는다.
- 밋밋한 마무리 대신 실천 가능한 조언이나 신선한 관점을 남겨 결론이 마지막 요점처럼 읽히고, 여운으로 닫는다.
- 확인하지 못한 것을 아는 것처럼 닫지 않고, 남은 질문은 `확인 필요`나 `더 공부할 것`으로 열어 둔다.

전체:

- 1인칭은 판단, 선택, 해석, 실제 경험을 표시할 때만 쓰인다.
- 한 글에 관통하는 메시지 하나가 남고, 제목이 H4까지 깊어지지 않는다.
- 불확실한 내용은 `확인 필요`, `더 공부할 것`, 또는 유보 표현으로 남긴다.

### 포맷

발행 가능한 글이면 아래를 확인한다. 원본 노트나 학습 메모는 실패 대신 개선 권고로 처리할 수 있다.

- 주제 별은 H1으로 시작하고 H2, H3 순서를 건너뛰지 않는다.
- 순서 없는 목록은 hyphen bullet을 사용한다.
- 코드 블록에는 언어 태그가 있고, 앞 설명과 뒤 해석이 있다.
- 본문은 한국어이고, 기술 용어와 명령어는 실무 영어 표기를 유지한다.
- 마지막에 `참고자료`가 있거나, 자료가 없으면 `확인 필요`로 표시한다.

## 종합 판정

- `통과`: hard failure가 없고 핵심 스타일이 대부분 충족된다.
- `통과하지만 개선 필요`: akbun 스타일은 유지되지만 포맷, 참고자료, 깊이 중 일부가 부족하다.
- `실패`: hard failure가 있거나, 정의/원리/판단 없이 일반 기술 문서처럼 보인다.

## 보고 형식

```markdown
## akbun 스타일 리뷰 결과

### 통과 항목

- [항목명]: 근거 한 줄

### 실패 항목

- [항목명]: 실패 이유와 수정 방향 한 줄

### 해당 없음

- [항목명]: 평가하지 않은 이유

### 종합 판정

- akbun 스타일 여부: **통과** / **통과하지만 개선 필요** / **실패**
- 핵심 판단 근거: 1-2문장
- hard failure: 있음/없음
```

## 자기 검증

리뷰 기준은 akbun의 실제 글을 과하게 떨어뜨리면 안 된다. 원본 글에서 보이는 임시 작성 흔적까지 실패로 처리하지 말고, 새로 생성한 발행 초안에 적용해야 할 기준과 원본 노트의 흔적을 구분한다.
