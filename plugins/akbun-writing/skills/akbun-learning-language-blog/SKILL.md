# 언어공부 일기형 정리

영어공부 또는 일본어공부 노트를 개인 일기형 학습 기록으로 정리한다. 원본 노트는 수정하지 않고, 에이전트 작업 공간의 현재 디렉터리에 새 Markdown 초안을 만든다.

## 작업 흐름

1. 입력 노트를 읽고 프런트매터와 본문을 분리한다.
2. 공부 언어를 판단한다: 영어 표현이 중심이면 영어공부, 일본어 표현이 중심이면 일본어공부로 쓴다. 둘 다 섞여 있으면 사용자에게 어느 언어 글로 정리할지 묻는다.
3. 공부 날짜를 정한다.
   - 프런트매터의 `created` 날짜를 우선 사용한다.
   - `created`가 없으면 파일명에 있는 날짜 또는 주차를 사용한다.
   - 둘 다 없으면 사용자에게 공부 날짜를 묻는다.
4. 원본의 단어, 표현, 예문, 헷갈린 문장을 분류한다.
5. `references/language-diary-template.md` 구조로 일기형 초안을 작성한다.
6. `references/language-diary-style.md` 기준으로 자연어 검토를 한 번 수행한다.
7. 결과를 에이전트 작업 공간의 현재 디렉터리에 `{source-stem}-blog.md`로 저장한다. 예: `2026년 5월 2주차 영어공부-blog.md`.

## 출력 규칙

- 원본 노트 파일은 수정하지 않는다.
- 기본 출력 위치는 현재 작업 디렉터리다. 입력 파일 옆에 저장하지 않는다.
- 출력 위치가 애매하거나 입력 파일이 여러 개면 사용자에게 묻는다.
- 글은 개인 일기형 학습 기록처럼 쓴다.
- `요약`, `목차`, `마무리`, `참고자료` 섹션은 쓰지 않는다.
- 회사명, 학원명, 수업명, 특정 기관명은 제목, 본문, 태그에 드러내지 않는다. 원본에 그런 이름이 있으면 `수업`, `공부 시간`, `학습 노트`처럼 일반 표현으로 바꾼다.
- 원본의 영어/일본어 표현은 보존한다. 한국어 설명은 자연스럽게 다듬는다.
- 보이지 않는 제어문자나 깨진 공백은 출력에서 조용히 제거한다.
- 발음, 직독직해, 문법 강의처럼 새 설명을 길게 추가하지 않는다. 그런 요청은 언어 학습 해설 스킬의 역할이다.

## 자연어 검토

초안을 저장하기 전에 아래 항목을 눈으로 점검한다.

- 요청이 영어공부/일본어공부 정리인지 확인했는가?
- 공부 날짜가 본문 상단에 들어갔는가?
- 첫 문단이 일기처럼 자연스럽고, 교재 요약문처럼 딱딱하지 않은가?
- 오늘 배운 표현, 헷갈렸던 표현, 다시 써보고 싶은 문장이 구분되는가?
- 회사명, 학원명, 수업명, 특정 기관명이 노출되지 않았는가?
- `요약`, `목차`, `마무리`, `참고자료` 섹션을 쓰지 않았는가?
- 결과 파일이 에이전트 작업 공간의 `{source-stem}-blog.md`에 저장되는가?
