# 해설 영상 스타일 토큰

수학·기술 해설 영상의 그림 언어를 고정하는 규칙이다. 주제가 무엇이든 이 토큰을 그대로 쓴다.

## 캔버스와 배경

- 기본 해상도 1920x1080. 세로 영상 요청 시 1080x1920.
- 배경은 아주 어두운 차콜 남색 `#0d1117` 단색. 그라데이션, 패턴, 사진 배경 금지.
- 콘텐츠는 상하좌우 최소 80px 여백 안에 배치한다.

## 색 팔레트

역할별로 색을 고정한다. 한 장면에 포인트 색은 2~3개까지만 쓴다.

| 역할 | 색 |
|---|---|
| 본문 텍스트 | `#e6edf3` |
| 보조 텍스트·캡션 | `#8b949e` |
| 주 개념(제목, 핵심 도형 테두리) | `#79c0ff` / `#58a6ff` |
| 관계·화살표·흐름 | `#f778ba` |
| 수식·강조·결과 | `#ffa657` |
| 보조 관계·부가 요소 | `#d2a8ff` |
| 성공·정답 표시 | `#7ee787` |

도형 채움은 테두리 색의 반투명(alpha 0.10~0.15)으로 만든다. 예: `rgba(88, 166, 255, 0.12)`.

## 타이포그래피

- 본문·제목: sans-serif(Inter 등). 제목 60~80px bold, 보조 문장 32~40px, 캡션 24~28px.
- 수식·코드: monospace(JetBrains Mono 등) 40~52px. 수식 기호는 유니코드(ᵀ, √, ·, Σ)로 쓴다.
- 장면당 텍스트는 제목급 1개 + 보조 문장 1개 이하. 문단 단위 텍스트 금지.

## 도형과 다이어그램

- 기본 도형은 rounded box(radius 12~20px, 테두리 3px, 반투명 채움)다.
- 연결선·화살표는 SVG `path`의 Q 곡선(부드러운 호)으로 그린다. 직선 화살표 남발 금지.
- 점선(`stroke-dasharray`)은 보조·간접 관계에만 쓴다.
- 배열·수열은 같은 크기 box를 등간격으로 나열하고, 강조할 원소만 테두리 색을 바꾼다.

## 모션 언어

- 요소는 한꺼번에 등장하지 않는다. 읽는 순서대로 0.3~0.5초 간격을 두고 순차 등장한다.
- 텍스트 등장: `opacity 0 -> 1` + `y` 20~40px 이동, 0.6~0.8초.
- 도형 등장: `opacity 0 -> 1` + `scale 0.6 -> 1`, 0.4~0.6초.
- 선·화살표 등장: `stroke-dashoffset`을 path 길이에서 0으로 줄여 "그려지듯" 표현, 0.8~1.2초.
- 강조: 색 변경 또는 scale 1 -> 1.08 -> 1 펄스. 흔들기, 회전, bounce ease 금지.
- 퇴장·전환: fade out 0.4초. 컷 전환 대신 fade를 쓴다.
- ease는 `power2.out` 계열로 통일한다.

## snapshot 검증 체크리스트

- 배경이 `#0d1117` 단색인가?
- 포인트 색이 역할 규칙대로 쓰였고 한 장면에 3개 이하인가?
- 장면에 핵심 문장이 1개뿐인가? 텍스트가 문단이 되지 않았는가?
- 텍스트·도형이 여백(80px) 안에 있고 서로 겹치지 않는가?
- 등장 순서가 읽는 순서(제목 -> 도형 -> 관계 -> 수식)와 일치하는가?
- 수식·코드가 monospace인가?
