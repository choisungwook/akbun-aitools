# akbun 외부 발표자료 생성 (라이트 샌드위치)

## 이 skill이 하는 일

아무 주제(개념·장애 경험·실습·코드)를 akbun 외부 발표 스타일의 슬라이드 덱으로 만든다.
밋업·세미나·컨퍼런스 등 외부 청중을 위한 형식으로, 다크 표지·섹션이 흰 내용 슬라이드를 감싸는
"라이트 샌드위치" 구조에 목차·실습자료 안내·페이지 번호·링크 푸터까지 갖춘 정식 발표 형식이다.

사내 스터디·내부 공유·블로그 보조용의 미니멀 다크 덱이 필요하면 이 skill 대신
**akbun-presentation-study** skill을 쓴다.

산출물은 두 가지다.

1. **슬라이드 아웃라인** — 슬라이드별 유형·헤더·내용 구성을 정리한 표. 생성 전 확인용.
2. **.pptx 파일** — pptxgenjs로 생성한 발표 파일. PowerPoint로 열거나 Google Slides로 가져간다.

## 스타일은 design.md를 따른다

**색·타이포·레이아웃·그림 언어·말투·스크린샷 규칙 등 "결과물이 어떻게 생겼는가"는 모두
[design.md](design.md)에 정의돼 있다.** 이 SKILL.md는 스킬 동작(작업 순서·QA·출력)만 다룬다.
발표를 만들기 전에 design.md를 읽고 스펙을 그대로 따른다.

design.md는 특정 도구에 의존하지 않는 자기완결적 스타일 스펙이라 다른 곳에 복붙해 재사용할 수
있다. pptxgenjs로 그 스펙을 구현한 코드 키트는 [references/pptxgenjs-kit.md](references/pptxgenjs-kit.md)에 있다.

## 작업 순서

1. **design.md 숙지.** [design.md](design.md)에서 색·타이포·레이아웃·그림 언어를 읽는다.
2. **주제 파악.** 입력에서 "핵심 질문 하나"를 뽑는다. 이 질문이 전환(훅) 슬라이드가 된다.
3. **아웃라인 작성.** design.md의 덱 구성 순서로 슬라이드별 유형·헤더·내용 구성을 표로 만든다.
4. **아웃라인 확인.** 사용자에게 보여주고 진행한다(피드백이 있으면 반영).
5. **pptx 생성.** [references/pptxgenjs-kit.md](references/pptxgenjs-kit.md)의 검증된 스타일
   키트를 읽고 그대로 복사해 시작한다. 좌표를 잡을 때 라벨·마커가 겹치지 않는지 계산한다.
6. **QA.** 파일을 렌더링해 텍스트 잘림·겹침·색 문법 위반을 확인하고 고친다(키트 문서의
   `macOS에서 QA` 참고).
7. **출력.** 아웃라인 + .pptx 경로 + 한국어 한 줄 설명을 출력한다.

## Google Slides로 가져가기

1. 생성한 .pptx를 Google Drive에 올린다.
2. 더블클릭 → "Google Slides로 열기"를 누르면 레이아웃이 유지된 채 변환된다.
3. Google Slides에는 나눔바른고딕이 없어 다른 고딕으로 대체된다. 글꼴이 중요하면 pptx를
   PowerPoint에서 그대로 쓰고, Google Slides는 공유·협업용으로만 쓴다.

## 예시 (gold reference)

입력 예:

```text
Redis 캐시 스탬피드를 밋업에서 발표할 자료 만들어줘
```

아웃라인(발췌):

| # | 유형 | 배경 | 내용 |
|---|---|---|---|
| 1 | 표지 | 다크 | "**Redis 캐시 스탬피드** 대응기"(키워드 노랑) + 발표자·이메일 |
| 2 | 목차 | 라이트 | 1. 캐시 스탬피드란? … 5. 마치며 |
| 3 | 실습자료 | 라이트 | 예제 repo 링크 + 실습 환경 안내 |
| 4 | 섹션 표지 | 다크 | 노란 세로 바 + "캐시 스탬피드란?" |
| 5 | 내용 | 라이트 | 헤더 "1. 캐시 스탬피드란?" + ■ 통신순서 ①②③ + 코드 패널(왼쪽) ➡ 다이어그램(오른쪽) + github 푸터 |
| 6 | 전환 | 라이트 | "같은 키의 캐시가 / 동시에 만료되면 / 무슨 일이 생길까요?" (중앙 3줄) |
| 7 | 내용 | 라이트 | 원인 스텝 다이어그램 — 빨간 점선 화살표 + 빨간 X |
| … | | | |
| N-1 | 섹션 표지 | 다크 | "마치며" |
| N | 내용 | 라이트 | 요약 → 다음 공부 → 감사 인사, 마지막에 참고자료 |

같은 캐시 다이어그램이 5·7·해결책 슬라이드에서 반복되며 마커와 강조만 바뀐다.

## 완료 전 확인

스타일 항목(색 문법·레이아웃·그림 언어)의 상세 기준은 [design.md](design.md)에 있다. 아래는
스킬 실행 관점의 최종 점검이다.

- design.md의 스펙을 따랐는가?
- 질문/전환 슬라이드가 정답보다 먼저 나오는가? 흐름에 ①②③ 마커가 있는가?
  노랑=포인트·빨강=문제 문법을 지켰는가? 실습에 github/데모 링크를 달았는가?
  외부 자료에 `[ 출처: ... ]`를 달았는가? 같은 다이어그램을 반복해 장면을 진행했는가?
- 표지에 발표자·이메일이 있는가? 목차·실습자료 안내가 앞에 있는가? 섹션 표지에
  노란 세로 바가 있는가? 내용 헤더가 `N. 섹션명` 형식인가? 페이지 번호가 표지를 제외한 모든
  슬라이드에 있는가? 마치며가 요약→다음 공부→감사 순서인가? 참고자료 슬라이드가 마지막에 있는가?
- 렌더링 QA로 텍스트 잘림·겹침을 확인했는가?
- 아웃라인 + .pptx 경로 + 한국어 한 줄 설명을 출력했는가?
