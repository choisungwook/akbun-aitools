# akbun 외부 발표 디자인 가이드 (라이트 샌드위치)

akbun 외부 발표자료의 스타일을 재현하기 위한 자기완결적 스펙이다. 색·타이포·레이아웃·그림 언어·
말투·스크린샷 규칙을 담는다. 특정 도구(pptxgenjs 등)에 의존하지 않으므로, 다른 프롬프트·문서·디자인
도구에 그대로 복붙해 스타일을 재현할 수 있다.

밋업·세미나·컨퍼런스 등 외부 청중을 위한 형식이다. 다크 표지·섹션이 흰 내용 슬라이드를 감싸는
"라이트 샌드위치" 구조로, 목차·실습자료 안내·페이지 번호·링크 푸터까지 갖춘 정식 발표 형식이다.
사내 스터디·내부 공유용 다크 스텝 형식은 akbun-presentation-study skill의 design.md를 따른다.

## 원칙

- **캔버스**: 16:9 와이드(13.33 × 7.5 in).
- **스토리는 "질문 → 여정 → 답"이다.** 정답을 먼저 말하지 않고, 질문/전환 슬라이드로 궁금증을
  만든 뒤 스텝을 밟아 답에 도착한다.
- **전환 슬라이드가 발표의 리듬이다.** 섹션 사이, 실습 직전, 반전 지점에 글자만 있는 전환
  슬라이드를 끼워 청중의 호흡을 조절한다.
- **순서는 숫자 마커로 밟는다.** 다이어그램의 흐름은 ①②③ 서클 숫자로 표시하고, 청중은 마커
  순서대로 그림을 읽는다.
- **강조 색 문법**: 노랑(`#FFC000`)은 포인트(섹션 바·표지 키워드·주인공), 빨강(`#FF0000`)은
  문제·에러·주목할 흐름. 한 슬라이드에 강조는 1~2개면 충분하다.
- **실습은 재현 가능해야 한다.** 실습 슬라이드에는 `github 링크: URL`, 데모 영상이 있으면
  `유투브 데모링크: URL`을 하단에 단다.
- **출처를 밝힌다.** 가져온 이미지·자료 아래 `[ 출처: ... ]` 캡션을 단다.
- **같은 다이어그램을 반복하며 장면을 진행한다.** 슬라이드마다 새 그림을 그리지 말고, 같은
  구도에서 마커·강조·설명만 바꿔 이야기가 진행되는 효과를 낸다.
- 영어 기술 용어(pod, iptables, TTL 등)는 번역하지 않고 그대로 쓴다.

## 말투

- **내용 슬라이드는 평어체 단정문·명사형**으로 짧게 쓴다: "~이다", "~한다", "설정방법", "통신순서".
- **전환·질문 슬라이드는 청중에게 말을 건다.** 해요체·합쇼체를 허용한다:
  "왜 통신이 안될까요?", "직접 디버깅해봐요!", "저희는 ~를 다룹니다", "자! 이제 ~를 다뤄봅시다".
- **시나리오로 구체화한다**: "시나리오: pod 1개가 분당 처리할 수 있는 요청 수 1,000".
- 질문에 답할 때는 **"정답: ~"** 또는 **"첫 번째 포인트: ~"** 로 시작한다.
- **꼬리지식 패턴**: 정의가 와닿지 않으면 배경지식으로 파고들었다가 "다시 돌아와서 ~란?"으로 복귀한다.
- **솔직하게 수준을 밝힌다**: "전문 분야가 아니라 겉핥기 수준으로 정리" 같은 고백을 아끼지 않는다.
- 결론은 **청유형**으로 닫는다: "~하자", "~까지 설계하자".
- 이모지는 덱 전체에서 0~1회.

## 색

| 역할 | 색 | 쓰는 곳 |
|---|---|---|
| 다크 배경 | `#212022` | 표지·섹션 표지 |
| 라이트 배경 | `#FFFFFF` | 내용·전환·목차 등 나머지 전부 |
| 본문 | `#000000` | 라이트 슬라이드의 텍스트·도형선 |
| 포인트 | `#FFC000` | 섹션 표지 세로 바, 표지 제목의 핵심 키워드 |
| 문제/강조 | `#FF0000` | 빨간 화살표·박스·X·경고 텍스트 |
| 워크로드 채움 | `#FFF2CC` | 다이어그램의 pod·앱 등 워크로드 박스 |
| 리소스 채움 | `#E2EFDA` | service 등 쿠버네티스/인프라 리소스 박스 |
| 그룹 채움 | `#F2F2F2` | Endpoints 같은 묶음 영역 |
| 코드 패널 | 배경 `#1E1E1E`, 기본 `#D4D4D4`, 키 `#569CD6`, 문자열 `#CE9178` | VS Code Dark+ 스타일 코드/YAML |

## 타이포그래피

본문은 **나눔바른고딕**(없으면 맑은 고딕), 코드는 **Consolas**. bold는 헤더·섹션 제목·라벨에 쓴다.

| 요소 | 크기 |
|---|---|
| 표지 제목 | 40pt bold (핵심 키워드만 노랑, 나머지 흰색) |
| 섹션 표지 제목 | 40pt bold 흰색 |
| 전환/질문 멘트 | 32pt bold 검정, 중앙, 2~4줄 스택 |
| 내용 헤더 | 18pt bold, `N. 섹션명` 형식 |
| ■ 메인 불릿 | 14pt |
| - 서브 불릿·본문 | 12pt |
| 다이어그램 라벨 | 10~11pt |
| 코드 패널 | 10pt Consolas |
| 푸터 링크·출처 캡션 | 9~10pt |
| 페이지 번호 | 11pt, 오른쪽 아래 (표지 제외 모든 슬라이드) |

## 덱 구성 (스토리 순서)

1. **표지** (다크) — 제목 중앙(핵심 키워드 노랑), 오른쪽 아래 `발표자: 이름` + 이메일
2. **목차** (라이트) — 번호 목록으로 섹션 나열
3. **실습자료 안내** (라이트) — 예제 github repo, 실습 환경(vagrant 등), 출처
4. **섹션 반복** ×N — 섹션마다 아래 리듬을 반복한다
   - 섹션 표지 (다크 + 노란 세로 바)
   - 개념 설명 (■ 불릿 + 다이어그램)
   - 전환 질문 ("왜 ~일까요?")
   - 원리/정답 (①②③ 스텝 다이어그램)
   - 실습 (코드 패널 + 링크 푸터), 디버깅 확인 (스크린샷)
5. **마치며** (섹션 표지 후 라이트) — 내용 요약 → 다음 공부 → 감사 인사
6. **참고자료** (라이트) — 번호 목록 링크

## 레이아웃 상세

좌표 단위는 inch, 원점은 슬라이드 왼쪽 위.

- **표지**: 제목 (0.9, 2.2) 폭 11.5 중앙 정렬. 발표자 정보는 (7.4, 5.4) 오른쪽 정렬.
- **섹션 표지**: 노란 세로 바 (0.42, 세로 중앙, 0.18 × 1.0). 제목은 바 오른쪽 (0.95)에서 시작.
  부제는 제목 아래 `- 부제` 20pt 흰색.
- **내용**: 헤더 (0.42, 0.2). 불릿 블록 (0.79, 0.96)에서 시작, ■ → `-` → ①②③ 들여쓰기 계층.
  불릿은 2~4줄까지만 쓰고 나머지는 다이어그램·코드가 말하게 한다.
  본문 영역 y 1.7~6.9에 다이어그램 하나(+코드 패널 하나) 배치.
- **실습/코드**: 코드 패널을 왼쪽(폭 3.5~4.5), 다이어그램을 오른쪽에 두고 사이에 굵은 검정
  화살표(➡)를 넣는 배치가 기본. 코드가 주인공이면 중앙 배치.
- **전환/질문**: 빈 흰 화면 중앙에 멘트만. 줄을 짧게 끊어 2~4줄로 쌓는다.
- **푸터**: 링크는 왼쪽 아래 (0.3, 7.12), 페이지 번호는 오른쪽 아래 (12.6, 7.05).

## 그림 언어 (라이트 다이어그램)

- **컴퍼넌트 박스**: 흰 채움 + 검정 0.75pt 테두리 + 검정 라벨. 필요하면 살짝 둥근 모서리.
- **워크로드/리소스 구분 채움**: pod·앱 등 워크로드는 연노랑(`#FFF2CC`), service 같은
  쿠버네티스·인프라 리소스는 연초록(`#E2EFDA`), 묶음 영역은 연회색(`#F2F2F2`).
- **논리 경계**: 노드·클러스터·호스트는 둥근 모서리 얇은 검정 테두리 박스. 라벨은 경계 안쪽
  왼쪽 위. 공식 로고(kubernetes 등)는 경계 왼쪽 위 모서리에 걸쳐 배치.
- **화살표**: 데이터 흐름은 검정 실선 + 삼각 화살촉. 설정·관리 흐름이나 주목할 경로는
  **빨간 점선 화살표**. 문제 지점은 빨간 X·빨간 박스.
- **순서 마커**: ①②③ 서클 숫자 문자를 라벨 앞이나 흐름 위에 단다.
- **텍스트 강조**: 본문 키워드는 노란 형광펜(하이라이트) 또는 빨간 글씨. 한 슬라이드 한 곳.
- **읽는 방향**: 왼쪽 → 오른쪽, 요청 → 응답.

## 스크린샷 규칙

- 터미널·콘솔 캡처는 **다크 테마 그대로**, 필요한 부분만 잘라 넣는다. 창 프레임은 잘라낸다.
- 봐야 할 줄에 **빨간 화살표, 빨간 밑줄, 빨간 박스 중 하나**를 딱 한 곳 단다.
- 캡처 아래 `[ 설명 ]` 형식의 짧은 캡션을 달 수 있다.
- 기술 로고 아이콘은 외부에서 내려받지 않는다. 사용자가 보유한 로고를 붙여넣게 자리를 남기고,
  생성 단계에서는 도형+라벨로 대체한다.
