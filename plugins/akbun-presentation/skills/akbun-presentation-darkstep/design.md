# akbun 다크 스텝 디자인 가이드

akbun 다크 스텝 발표자료의 스타일을 재현하기 위한 자기완결적 스펙이다. 색·타이포·레이아웃·그림 언어·
말투·스크린샷 규칙을 담는다. 특정 도구(pptxgenjs 등)에 의존하지 않으므로, 다른 프롬프트·문서·디자인
도구에 그대로 복붙해 스타일을 재현할 수 있다.

전 슬라이드 다크에 슬라이드당 메시지 한 줄만 싣는 미니멀 형식이다. 다크 표지·섹션이 흰 내용
슬라이드를 감싸는 라이트 샌드위치 형식은 akbun-presentation-lightsandwich skill의 design.md를
따른다.

## 원칙

- **캔버스**: 16:9 와이드(13.33 × 7.5 in).
- **스토리는 "질문 → 여정 → 답"이다.** 정답을 먼저 말하지 않고, 질문/훅 슬라이드로 궁금증을
  만든 뒤 스텝을 밟아 답에 도착한다.
- **훅/섹션 슬라이드가 발표의 리듬이다.** 섹션 사이, 반전 지점에 글자만 있는 슬라이드를 끼워
  청중의 호흡을 조절한다.
- **순서는 숫자 마커로 밟는다.** 다이어그램의 흐름은 숫자 마커로 표시하고, 청중은 마커
  순서대로 그림을 읽는다.
- **강조 색 문법**: 노랑(`#FFC000`)은 포인트(주인공·강조), 빨강(`#FF0000`)은 문제·에러·주목할
  흐름. 한 슬라이드에 강조는 1~2개면 충분하다.
- **실습은 재현 가능해야 한다.** 실습 내용에는 `github 링크: URL`, 데모 영상이 있으면
  `유투브 데모링크: URL`을 단다.
- **출처를 밝힌다.** 가져온 이미지·자료 아래 `[ 출처: ... ]` 캡션을 단다.
- **같은 다이어그램을 반복하며 장면을 진행한다.** 슬라이드마다 새 그림을 그리지 말고, 같은
  구도에서 마커·강조·설명만 바꿔 이야기가 진행되는 효과를 낸다.
- 영어 기술 용어(pod, iptables, TTL 등)는 번역하지 않고 그대로 쓴다.

## 말투

- **내용 슬라이드는 평어체 단정문·명사형**으로 짧게 쓴다: "~이다", "~한다", "설정방법", "통신순서".
- **훅/질문 슬라이드는 평어체 질문**을 쓴다: "왜 문제가 생겼을까?".
- **시나리오로 구체화한다**: "시나리오: pod 1개가 분당 처리할 수 있는 요청 수 1,000".
- 질문에 답할 때는 **"정답: ~"** 또는 **"첫 번째 포인트: ~"** 로 시작한다.
- **꼬리지식 패턴**: 정의가 와닿지 않으면 배경지식으로 파고들었다가 "다시 돌아와서 ~란?"으로 복귀한다.
- **솔직하게 수준을 밝힌다**: "전문 분야가 아니라 겉핥기 수준으로 정리" 같은 고백을 아끼지 않는다.
- 결론은 **청유형**으로 닫는다: "~하자", "~까지 설계하자".
- 이모지는 덱 전체에서 0~1회.

## 색과 타이포그래피

전 슬라이드 배경 진회색 `#252525` 단색. 색은 5개로 제한한다: 배경 `#252525`, 본문 `#EBEBEB`,
도형선 `#FFFFFF`, 포인트 `#FFC000`, 문제 `#FF0000`. 폰트는 **G마켓 산스 Medium** 하나만
쓴다(무료 배포 폰트, 없으면 Pretendard).

| 요소 | 크기 |
|---|---|
| 표지 제목 | 48~54pt bold 흰색 |
| 훅(질문)·섹션 슬라이드 | 36~44pt 중앙 |
| 슬라이드 헤더(덱 주제 반복) | 28pt, 위치 (0.46, 0.39) 고정 |
| 메시지 한 줄 | 16pt 불릿, (0.75, 1.0) |
| 다이어그램 라벨 | 11~14pt |
| 참고자료·캡션 | 10pt |

## 슬라이드 유형 6종

1. **표지**: 왼쪽 정렬 제목 + 부제 한 줄.
2. **훅/섹션**: 빈 화면 중앙에 질문 또는 섹션명 한두 줄. 다른 요소 없음.
3. **내용**: 헤더에 덱 주제를 같은 자리·같은 문구로 반복 + 메시지 불릿 **한 줄** + 본문
   다이어그램 하나. **말할 게 두 개면 슬라이드를 나눈다.**
4. **스크린샷**: 내용형 + 터미널 캡처.
5. **코드/명령어**: 내용형 + 얇은 흰 테두리 명령어 박스(폭 55~70%, 16pt) 하나.
6. **마치며**: 결론 한 줄 + 왼쪽 아래 `참고자료: URL` 10pt.

덱 마지막에 **아이콘 팔레트 슬라이드**(헤더 "아이콘")를 두는 습관을 따른다. 자주 쓰는 도형
견본과 기술 로고를 모아두는 작업용 슬라이드로, 발표 때는 숨긴다.

## 그림 언어 (다크 다이어그램)

- **컴퍼넌트 박스**: 채움 없음 + 흰 1pt 테두리 + `#EBEBEB` 라벨. 역할 이름을 소문자로 짧게.
- **주인공 박스**: 슬라이드가 설명하는 컴퍼넌트 하나만 노랑 채움 + 검정 bold 라벨.
- **논리 경계**: 점선 흰 사각형, 라벨은 경계선 바로 위 왼쪽.
- **화살표**: 흰 실선 1.5pt + 삼각 화살촉, 라벨은 화살표 위. 문제 흐름은 화살표·라벨 모두 빨강.
- **숫자 마커**: 지름 0.3in 채움 원. 정상 스텝은 노랑 원 + 검정 숫자, 문제 스텝은 빨강 원 + 흰 숫자.
- **문제 표시**: 빨강 `✕` + 아래 빨강 캡션 한 줄.
- **사람/스토리텔링**: 사람 실루엣 아이콘 + 흰 말풍선 + 검정 대사.
- **밀도**: 슬라이드당 다이어그램 하나, 박스 5~7개 이하, 여백 넉넉히.
- **읽는 방향**: 왼쪽 → 오른쪽, 요청 → 응답.

## 스크린샷 규칙

- 터미널·콘솔 캡처는 **다크 테마 그대로**, 필요한 부분만 잘라 넣는다. 창 프레임은 잘라낸다.
- 봐야 할 줄에 **빨간 화살표, 빨간 밑줄, 빨간 박스 중 하나**를 딱 한 곳 단다.
- 캡처 아래 `[ 설명 ]` 형식의 짧은 캡션을 달 수 있다.
- 기술 로고 아이콘은 외부에서 내려받지 않는다. 사용자가 보유한 로고를 붙여넣게 자리를 남기고,
  생성 단계에서는 도형+라벨로 대체한다.
